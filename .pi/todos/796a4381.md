Implemented session delete flow in frontend with confirmation and React Query mutation wiring, plus backend delete endpoint support.

## What changed

### Frontend
- `frontend/src/renderer/src/api/conversationsApi.ts`
  - Added `deleteConversationSession(conversationId)` calling `DELETE /conversation/{conversationId}`.
- `frontend/src/renderer/src/queries/conversationsQueries.ts`
  - Added `useDeleteConversationSessionMutation()`.
  - Uses mutation pattern and updates list cache immediately via `setQueriesData`.
  - Invalidates conversation list queries and removes deleted detail query.
- `frontend/src/renderer/src/components/ConversationHistorySidebar.tsx`
  - Added per-session **Delete** action in the session list.
  - Added confirmation step via `window.confirm(...)`.
  - Added loading disabled state and error/success toasts.
- `frontend/src/renderer/src/components/SessionTranscription.tsx`
  - Added **Delete session** action in conversation detail context.
  - Added confirmation and mutation handling.
  - Navigates user back to home after successful deletion.
- `frontend/src/renderer/src/assets/main.css`
  - Added styles for sidebar delete affordance and conversation-detail delete button.

### Backend (required wiring)
- `backend/api/routes/conversation.py`
  - Added `DELETE /conversation/{conversation_id}` endpoint.
  - Deletes session messages, documents, SQL chunk rows, Chroma chunk ids, local uploaded files, and session row.
  - Returns structured deletion counts.

### Schema
- `frontend/src/renderer/src/schemas/conversation.ts`
  - Added `DeleteConversationResponse` type.

## Validation
- Ran frontend type checks successfully:
  - `cd frontend && npm run typecheck`

## Acceptance criteria mapping
- User can delete a session from UI with confirmation ✅
- UI removes deleted session (immediate cache update + invalidate) ✅
- Loading/disabled and error feedback handled ✅
