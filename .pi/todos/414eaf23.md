Implemented a reusable session transcription component with in-session follow-up Q&A flow.

## Frontend
- Added new reusable component:
  - `frontend/src/renderer/src/components/SessionTranscription.tsx`
  - Renders session message history (user + agent turns)
  - Includes follow-up question form bound to current session id
  - Handles pending + error states for follow-up requests
- Updated session route:
  - `frontend/src/renderer/src/routes/ConversationRoute.tsx`
  - Uses `SessionTranscription` when session data is loaded
  - Preserves clear loading/invalid/error states
- Extended conversation API:
  - `frontend/src/renderer/src/api/conversationsApi.ts`
  - Added `askFollowUpQuestion(conversationId, question)`
- Extended query hooks:
  - `frontend/src/renderer/src/queries/conversationsQueries.ts`
  - Added `useAskFollowUpQuestionMutation(conversationId)`
  - Invalidates session detail + conversation list on success
- Added response schema:
  - `frontend/src/renderer/src/schemas/conversation.ts`
  - `FollowUpQuestionResponse`
- Added styling for follow-up input/actions:
  - `frontend/src/renderer/src/assets/main.css`

## Backend
- Added follow-up endpoint:
  - `POST /api/v1/conversation/{conversation_id}/ask`
  - File: `backend/api/routes/conversation.py`
- Behavior:
  - Validates non-empty question
  - Loads existing session history
  - Retrieves context (`retrieve_context`) + generates response (`generate_agent_response`)
  - Persists both user follow-up message and agent response to the same session
  - Returns structured follow-up payload with retrieved context count

## Validation
- Backend compile check passed:
  - `python -m compileall backend/api/routes/conversation.py`

  - `cd frontend && npm run typecheck`
