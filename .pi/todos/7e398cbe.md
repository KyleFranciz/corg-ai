Update the listening screen so the user's spoken question appears visibly while/after capture, allowing users to confirm what they asked.

## Goals
- Display the user's question text on the listening view.
- Keep it readable and consistent with current UI style.
- Preserve behavior across listening -> thinking -> response transitions.

## Acceptance criteria
- During listening, user sees their captured question text (when available).
- If no transcript is available yet, UI handles empty state gracefully.
- The shown question persists into subsequent pipeline states as appropriate.
- No regressions to existing mic/listening interactions.
