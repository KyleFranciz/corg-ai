Implement and verify true streaming behavior in the voice pipeline so responses begin as soon as content is generated.

Scope:
- Stream LLM output tokens/chunks instead of waiting for full completion.
- Segment streamed text into sentence-safe chunks for TTS.
- Stream/queue audio generation and playback so speech starts before full response is complete.
- Ensure ordering guarantees (no overlaps, no dropped chunks) and interruption safety.
- Emit real-time websocket events for streaming states/progress.

Acceptance criteria:
- User hears first spoken sentence quickly after LLM generation begins.
- No full-response blocking before TTS starts.
- Web UI receives incremental response/audio progress updates.
- Works fully offline with local model + local TTS stack.
