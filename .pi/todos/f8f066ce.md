Update the system prompt so the agent does not greet the user repeatedly on each iteration/turn.

## Goals
- Prevent repetitive greetings across ongoing conversations.
- Keep tone natural and concise after initial interaction.
- Preserve ability to greet appropriately at session start when needed.

## Acceptance criteria
- Agent avoids greeting on every response turn.
- Greeting behavior is context-aware (e.g., initial turn vs follow-up turns).
- Prompt instructions are clear and do not conflict with other response-style rules.
- Verified in representative multi-turn chat flows.
