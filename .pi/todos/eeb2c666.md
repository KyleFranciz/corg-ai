Update the Start Audio button behavior so clicking it again cancels/stops recording/streaming, making it a true start/stop toggle.

## Goals
- Convert Start Audio control into a toggle.
- Support cancel/stop when button is clicked while audio is active.
- Keep UI state in sync with actual audio state.

## Acceptance criteria
- First click starts audio.
- Second click while active stops/cancels audio cleanly.
- Button label/state reflects current mode (start vs stop).
- No orphaned audio processes/streams remain after stopping.
- Errors are handled with clear user feedback.
