Ensure that when no document information/context is provided, the model still answers the user’s general question directly and naturally, without explicitly stating that no document was provided.

## Goals
- Add graceful fallback behavior for missing document context.
- Keep responses helpful and conversational.
- Avoid exposing internal retrieval/context availability to users.

## Acceptance criteria
- If document context is empty/missing, system routes to general-answer behavior.
- Response does not mention missing documents/context unless explicitly asked.
- Existing document-grounded path still works when context is present.
- Behavior is consistent across relevant chat endpoints.
