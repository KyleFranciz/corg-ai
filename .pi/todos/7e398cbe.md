Implemented support for showing the user's spoken question on the listening screen and preserving it through later pipeline states.

## What changed
- Backend (`backend/api/routes/websocket.py`)
  - Included full `transcript` text in `status` event details when `transcribing` stage completes.
- Frontend hook (`frontend/src/renderer/src/hooks/useAgent.ts`)
  - On `status` events, when stage=`transcribing` and state=`completed`, reads `details.transcript` and sets `transcript` immediately.
  - This makes the question appear before final `result` event, so it persists naturally into thinking/response UI states.
- Listening UI (`frontend/src/renderer/src/App.tsx`)
  - Updated listening screen to always render a transcript area.
  - Shows captured question when available; otherwise displays graceful placeholder text.
- Styling (`frontend/src/renderer/src/assets/main.css`)
  - Added `.corg-user-transcript-placeholder` style aligned with existing typography and muted color.
