Implement backend support for deleting a session by ID and removing all associated local data.

Tasks:
- Add route handler in backend API to delete a session by ID.
- Delete session-owned records (messages, metadata, attachments, and any mapped entities).
- Remove session-related vector entries from local vector store (CHROMA_PATH/local index) where applicable.
- Ensure operation is idempotent/safe for already-deleted IDs (clear 404 vs no-op behavior).
- Return structured response with deleted ID and cleanup summary.
- Add boundary error handling/logging for cleanup failures.

Acceptance criteria:
- API deletes session and all linked local artifacts.
- No orphaned DB/files/vector references remain.
- Endpoint returns consistent success/error responses.
