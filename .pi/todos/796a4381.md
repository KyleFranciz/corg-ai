Add a user-facing delete action for sessions and wire it to backend deletion.

Tasks:
- Add delete affordance in session list/detail context.
- Add confirmation dialog to prevent accidental deletions.
- Call delete-session API through renderer API layer.
- Use React Query mutation pattern (invalidate/refetch session queries after success).
- Handle loading/disabled states and surface errors clearly.
- Remove deleted session from visible UI state immediately after success.

Acceptance criteria:
- User can delete a session from UI with confirmation.
- UI updates to remove deleted session without stale entries.
- Error states are handled and communicated.
