Implemented end-to-end document upload + ingestion flow across frontend and backend.

## Backend
- Added new route: `POST /api/v1/documents/ingest` in `backend/api/routes/documents.py`.
- Route accepts multipart file uploads (`files` list), validates filename + extension (`.txt`, `.md`, `.pdf`), saves files locally under `CORG_DATA_DIR/documents`, and ingests each file into local storage/Chroma via existing ingestion logic.
- Returns structured response including uploaded filenames, failed files, and ingestion summary counters.
- Registered router in `backend/api/router.py`.

## Frontend
- Added API client module: `frontend/src/renderer/src/api/documentsApi.ts` with multipart upload function.
- Added React Query mutation hook: `frontend/src/renderer/src/queries/documentsQueries.ts`.
- Wired upload UI in `frontend/src/renderer/src/App.tsx`:
  - hidden `<input type="file" multiple accept=".txt,.md,.pdf">`
  - upload button triggers file picker
  - selected files are posted to backend ingest endpoint
  - basic success/error feedback shown in UI

## Validation
- Backend syntax check: `python -m compileall backend/api/routes/documents.py backend/api/router.py`
- Frontend typecheck: `cd frontend && npm run typecheck` (passes)
