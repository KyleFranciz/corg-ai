Set up document upload functionality in the frontend and send uploaded files to the backend for ingestion and storage.

## Goals
- Add/complete frontend document upload flow.
- Transmit selected files from frontend to backend.
- Ensure backend exposes a route to receive and ingest uploaded documents.
- Persist ingested documents in backend storage.

## Acceptance criteria
- User can select one or more documents in the frontend UI.
- Frontend sends files to backend via a defined API endpoint.
- Backend route validates inputs and handles file ingestion.
- Backend stores documents locally (offline-first) and returns clear success/error responses.
- End-to-end flow is wired and callable from the app.
