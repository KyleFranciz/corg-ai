Ensure documents uploaded/shared in a conversation session are saved and associated with that specific session so users can continue asking follow-up questions against the same document set.

## Goals
- Associate uploaded documents with a conversation session.
- Reuse that session’s document context for subsequent questions.
- Preserve offline-first local storage behavior.

## Acceptance criteria
- Documents shared during a session are stored with session linkage.
- Follow-up questions in that session retrieve context from those same documents.
- Starting a new session does not automatically mix prior session document context (unless explicitly intended).
- Session history can indicate which documents were attached/shared.
- Clear handling for missing/removed files and ingestion failures.
