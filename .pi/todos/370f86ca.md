Implemented end-to-end streaming updates across backend WebSocket pipeline and frontend event handling.

## Backend changes
- Added LLM streaming generator in `backend/services/agent/chain.py`:
  - `stream_agent_response_chunks(...)` now streams model output chunks from Ollama via `llm.stream(...)`.
- Exported streaming API from `backend/services/agent/__init__.py`.
- Updated `backend/api/routes/websocket.py` pipeline:
  - Streams response chunks to client in real time with `type: response_chunk` events.
  - Segments text into sentence-safe chunks (`. ! ? \n`) for early TTS playback.
  - Queues TTS segments and plays in-order with a single worker (no overlaps / ordering preserved).
  - Sends `type: audio_progress` events as spoken segments complete.
  - Starts speaking before full response completion when first segment is ready.
  - Keeps final `result` event and full assembled response persistence in DB.

## Frontend changes
- Extended WebSocket event parser (`frontend/src/renderer/src/lib/websocket.ts`) with:
  - `response_chunk`
  - `audio_progress`
- Updated `useAgent` (`frontend/src/renderer/src/hooks/useAgent.ts`) to append streamed `response_chunk` content live into `response` state.

## Validation
- Backend compile check passed:
  - `python -m compileall backend/services/agent/chain.py backend/services/agent/__init__.py backend/api/routes/websocket.py`
- Frontend typecheck passed:
  - `cd frontend && npm run typecheck`

## Notes
- `result.audio_duration_seconds` now reports speaking duration timing (streaming playback duration) instead of recorded input duration.
- Agent message `audio_path` is currently persisted as `None` for streamed segmented playback.
