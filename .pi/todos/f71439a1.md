Add a dedicated UI state/page for when the agent is relaying its answer, and display the response text progressively in real time as speech is generated/played.

## Goals
- Introduce a clear "speaking" view/state in the frontend.
- Render the agent response live (token/chunk/segment streaming) instead of only showing final text.
- Keep transcript synchronized with audio playback so users can follow along.

## Acceptance criteria
- When agent transitions to speaking, UI switches to a speaking page/state.
- User sees response content appear incrementally while the agent is speaking.
- Final response remains visible after speaking completes.
- Error/interruption handling is graceful (partial text preserved when possible).
- Works offline with current local backend/WebSocket architecture.

## Notes
- Likely requires backend WebSocket message support for partial response chunks and frontend handling in `useAgent` + renderer components.
- Keep implementation aligned with existing pipeline status events and Sonner feedback.
